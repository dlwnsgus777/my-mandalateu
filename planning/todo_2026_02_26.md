## 발견된 버그

### 버그 1: `Alert.alert` 웹 환경 미동작

**증상**
- SettingsScreen의 로그아웃 버튼 클릭 시 확인 다이얼로그가 아무것도 나타나지 않음

**원인 분석 (Playwright로 확인)**
| 항목 | 결과 |
|------|------|
| DOM 클릭 이벤트 발화 | ✅ (`clickFired: true`) |
| `window.confirm()` 호출 여부 | ❌ (호출 안 됨) |
| `window.alert()` 호출 여부 | ❌ (호출 안 됨) |
| DOM에 Alert 렌더링 | ❌ (DOM에 없음) |
| `닉네임 변경` 모달(React state) | ✅ 정상 동작 |

**결론**: React Native의 `Alert.alert()`이 Expo web 환경에서 완전히 silent fail.
`window.confirm/alert`을 호출하지도 않고, DOM에도 아무것도 렌더링하지 않음.

**같은 패턴으로 영향받는 코드**
- `handleLogout` → `Alert.alert('로그아웃', ...)`
- `handleReset` → `Alert.alert('데이터 초기화', ...)`
- `handleDeleteAccount` → `Alert.alert('회원 탈퇴', ...)`
