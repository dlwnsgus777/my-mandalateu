# My Mandalateu - 만다라트 TODO 애플리케이션

## 프로젝트 개요

만다라트(Mandalart) 기법을 활용한 TODO 리스트 관리 애플리케이션

### 만다라트란?
- 일본의 목표 설정 기법
- 3x3 그리드를 사용하여 중심 목표를 8개의 세부 목표로 분해
- 각 세부 목표를 다시 8개의 실행 과제로 세분화
- 총 64개의 구체적인 실행 항목 도출

## 기술 스택 결정

### 선택: React Native (네이티브 앱)

**선택 이유:**
- 한 번의 개발로 iOS/Android 모두 지원
- JavaScript/TypeScript 활용 가능
- 네이티브 성능과 UX 제공
- 웹보다 나은 오프라인 기능
- 푸시 알림, 위젯 등 네이티브 기능 활용 가능

**개발 방식:**
- Expo 프레임워크 사용 (추천)
  - 빠른 개발 및 테스트
  - 간편한 빌드 및 배포
  - 풍부한 내장 라이브러리
  - 필요시 eject 가능

## 핵심 기능

### 1. 만다라트 그리드 UI
- 3x3 메인 그리드 (중심 목표 + 8개 세부 목표)
- 각 세부 목표 클릭 시 3x3 상세 그리드 표시
- 직관적인 시각적 표현
- 터치 제스처 지원

### 2. TODO 관리
- 각 셀을 TODO 항목으로 관리
- 완료 체크 기능
- 우선순위 설정
- 마감일 설정 (선택사항)
- 진행률 표시

### 3. 데이터 관리
- 로컬 데이터 저장 (AsyncStorage 또는 SQLite)
- 여러 만다라트 프로젝트 관리
- 데이터 백업/복원 기능
- (추후) 클라우드 동기화

### 4. 추가 기능
- 다크모드 지원
- 푸시 알림 (마감일 알림)
- 통계 및 진행 현황 대시보드
- 카테고리/태그 시스템

## 프로젝트 구조 (예정)

```
my-mandalateu/
├── src/
│   ├── components/        # 재사용 가능한 컴포넌트
│   │   ├── MandalartGrid/ # 만다라트 그리드 컴포넌트
│   │   ├── TodoCell/      # 개별 셀 컴포넌트
│   │   └── common/        # 공통 컴포넌트
│   ├── screens/           # 화면 컴포넌트
│   │   ├── HomeScreen/
│   │   ├── MandalartDetail/
│   │   └── SettingsScreen/
│   ├── store/             # 상태 관리 (Zustand/Redux)
│   ├── services/          # 데이터 서비스
│   ├── utils/             # 유틸리티 함수
│   └── types/             # TypeScript 타입 정의
├── assets/                # 이미지, 폰트 등
└── App.tsx
```

## 기술 스택 상세

### 프론트엔드
- **React Native** - 메인 프레임워크
- **Expo** - 개발 플랫폼
- **TypeScript** - 타입 안정성

### 상태 관리
- **Zustand** 또는 **Redux Toolkit** (검토 필요)

### 네비게이션
- **React Navigation** - 화면 전환

### 데이터 저장
- **AsyncStorage** - 간단한 데이터
- **SQLite** (선택) - 복잡한 데이터 구조

### UI 라이브러리
- **React Native Paper** 또는 **Native Base** (검토 필요)
- 커스텀 컴포넌트 활용

### 추가 라이브러리
- **React Native Gesture Handler** - 제스처 처리
- **React Native Reanimated** - 애니메이션
- **Date-fns** - 날짜 처리
