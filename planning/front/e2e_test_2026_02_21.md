## TC-01. 온보딩 (Onboarding)

> 최초 실행 시(`isFirstLaunch = true`) 온보딩 화면이 표시됨

### TC-01-1. 온보딩 슬라이드 순서 확인
- **Given** 앱을 최초 실행한다
- **When** 화면이 로드된다
- **Then** 슬라이드 1 "만다라트란?" 이 표시된다

### TC-01-2. 다음 버튼으로 슬라이드 전환
- **Given** 온보딩 슬라이드 1이 표시된 상태
- **When** "다음" 버튼을 클릭한다
- **Then** 슬라이드 2 "세부 목표 8개" 가 표시된다
- **When** "다음" 버튼을 클릭한다
- **Then** 슬라이드 3 "실행 과제 64개" 가 표시된다
- **When** "다음" 버튼을 클릭한다
- **Then** 슬라이드 4 "시작할 준비가" 가 표시되고 버튼이 "시작하기" 로 변경된다

### TC-01-3. 페이지 인디케이터 활성 상태 변경
- **Given** 온보딩 화면이 표시된 상태
- **When** 슬라이드를 넘길 때마다
- **Then** 인디케이터 dot이 현재 슬라이드에 맞게 활성화된다

### TC-01-4. 마지막 슬라이드에서 건너뛰기 버튼 미표시
- **Given** 슬라이드 1~3에서는 "건너뛰기" 버튼이 표시된다
- **When** 슬라이드 4(마지막)로 이동한다
- **Then** "건너뛰기" 버튼이 사라진다

### TC-01-5. 건너뛰기 버튼 → Login 이동
- **Given** 온보딩 슬라이드 1이 표시된 상태
- **When** "건너뛰기" 버튼을 클릭한다
- **Then** 로그인 화면으로 이동한다

### TC-01-6. 시작하기 버튼 → Login 이동
- **Given** 마지막 슬라이드가 표시된 상태
- **When** "시작하기" 버튼을 클릭한다
- **Then** 로그인 화면으로 이동한다

## TC-02. 회원가입 (Signup)

### TC-02-1. 회원가입 화면 진입
- **Given** 로그인 화면이 표시된 상태
- **When** "회원가입" 링크를 클릭한다
- **Then** 회원가입 화면으로 이동하고 "회원가입" 타이틀이 표시된다

### TC-02-2. 빈 폼 제출 시 에러
- **Given** 회원가입 화면에서 아무것도 입력하지 않은 상태
- **When** "회원가입" 버튼을 클릭한다
- **Then** "모든 항목을 입력해주세요." 에러 메시지가 표시된다

### TC-02-3. 닉네임 1자 입력 시 에러
- **Given** 닉네임에 "A", 이메일과 비밀번호는 유효하게 입력한 상태
- **When** "회원가입" 버튼을 클릭한다
- **Then** "닉네임은 2자 이상이어야 합니다." 에러 메시지가 표시된다

### TC-02-4. 이메일 형식 오류 시 에러
- **Given** 닉네임/비밀번호는 유효하게, 이메일은 "notanemail" 입력한 상태
- **When** "회원가입" 버튼을 클릭한다
- **Then** "올바른 이메일 형식을 입력해주세요." 에러 메시지가 표시된다

### TC-02-5. 비밀번호 7자 입력 시 에러
- **Given** 닉네임/이메일은 유효하게, 비밀번호는 "1234567"(7자) 입력한 상태
- **When** "회원가입" 버튼을 클릭한다
- **Then** "비밀번호는 8자 이상이어야 합니다." 에러 메시지가 표시된다

### TC-02-6. 정상 회원가입 후 로그인 화면 이동
- **Given** 닉네임, 이메일, 비밀번호를 유효하게 입력한 상태 (백엔드 연동)
- **When** "회원가입" 버튼을 클릭한다
- **Then** 로그인 화면으로 이동한다

### TC-02-7. 중복 이메일 회원가입 시 에러
- **Given** 이미 가입된 이메일로 회원가입을 시도한다 (백엔드 연동)
- **When** "회원가입" 버튼을 클릭한다
- **Then** "이미 사용 중인 이메일입니다." 에러 메시지가 표시된다

### TC-02-8. 로그인 링크 이동
- **Given** 회원가입 화면이 표시된 상태
- **When** "로그인" 링크를 클릭한다
- **Then** 로그인 화면으로 이동한다

## TC-03. 로그인 (Login)

### TC-03-1. 로그인 화면 기본 요소 확인
- **Given** 로그인 화면으로 이동한다
- **Then** 로고(🎯), "MY MANDALATEU" 앱 이름, 이메일/비밀번호 입력 필드, "로그인" 버튼, "회원가입" 링크가 표시된다

### TC-03-2. 빈 폼 제출 시 에러
- **Given** 아무것도 입력하지 않은 상태
- **When** "로그인" 버튼을 클릭한다
- **Then** "이메일과 비밀번호를 입력해주세요." 에러 메시지가 표시된다

### TC-03-3. 잘못된 인증 정보 로그인 (백엔드 연동)
- **Given** 존재하지 않는 이메일/비밀번호를 입력한 상태
- **When** "로그인" 버튼을 클릭한다
- **Then** "이메일 또는 비밀번호가 올바르지 않습니다." 에러 메시지가 표시된다

### TC-03-4. 정상 로그인 후 홈 이동 (백엔드 연동)
- **Given** 유효한 이메일/비밀번호를 입력한 상태
- **When** "로그인" 버튼을 클릭한다
- **Then** 홈 화면으로 이동하고 만다라트 그리드가 표시된다

### TC-03-5. 회원가입 링크 이동
- **Given** 로그인 화면이 표시된 상태
- **When** "회원가입" 링크를 클릭한다
- **Then** 회원가입 화면으로 이동한다

## TC-04. 홈 화면 (Home)

### TC-04-1. 홈 화면 기본 요소 확인
- **Given** 로그인하여 홈 화면에 진입한다
- **Then** 프로젝트 카드, 전체 진행률, 3×3 블록 그리드, "통계 보기" 버튼, "+ 새 프로젝트" 버튼이 표시된다

### TC-04-2. 전체 진행률 표시
- **Given** 홈 화면이 표시된 상태
- **Then** "전체 진행률 · X/Y" 형태의 텍스트와 프로그레스 바, 퍼센트가 표시된다

### TC-04-3. 프로젝트 제목 편집 모달
- **Given** 홈 화면이 표시된 상태
- **When** 프로젝트 카드(제목 영역)를 클릭한다
- **Then** "프로젝트 이름 변경" 모달이 표시되고 현재 제목이 입력 필드에 채워진다
- **When** 새 이름을 입력하고 "저장" 버튼을 클릭한다
- **Then** 모달이 닫히고 프로젝트 카드의 제목이 변경된다

### TC-04-4. 프로젝트 제목 편집 취소
- **Given** 제목 편집 모달이 열린 상태
- **When** "취소" 버튼을 클릭한다
- **Then** 모달이 닫히고 기존 제목이 유지된다

### TC-04-5. 블록 탭 → BlockDetail 이동
- **Given** 홈 화면에 3×3 블록 그리드가 표시된 상태
- **When** 임의의 블록을 클릭한다
- **Then** 해당 블록의 상세 화면(BlockDetail)으로 이동한다

### TC-04-6. 블록 롱프레스 → 액션시트 표시
- **Given** 홈 화면이 표시된 상태
- **When** 임의의 블록을 롱프레스(길게 클릭)한다
- **Then** "제목 편집", "메모 작성", "색상 변경", "취소" 항목이 포함된 바텀시트가 표시된다

### TC-04-7. 액션시트 취소
- **Given** 롱프레스 액션시트가 표시된 상태
- **When** "취소" 버튼을 클릭한다
- **Then** 액션시트가 닫힌다

### TC-04-8. 블록 제목 편집 (롱프레스 → 제목 편집)
- **Given** 롱프레스 액션시트가 표시된 상태
- **When** "제목 편집"을 클릭한다
- **Then** "블록 제목 편집" 모달이 표시된다
- **When** 새 제목을 입력하고 "저장"을 클릭한다
- **Then** 블록 그리드에 변경된 제목이 반영된다

### TC-04-9. 통계 보기 버튼 → Dashboard 이동
- **Given** 홈 화면이 표시된 상태
- **When** "통계 보기" 버튼을 클릭한다
- **Then** 통계(Dashboard) 화면으로 이동한다

### TC-04-10. 새 프로젝트 버튼 → CreateProject 이동
- **Given** 홈 화면이 표시된 상태
- **When** "+ 새 프로젝트" 버튼을 클릭한다
- **Then** 새 프로젝트 생성 화면으로 이동한다

## TC-05. 블록 상세 (BlockDetail)

### TC-05-1. 블록 상세 화면 기본 요소 확인
- **Given** 홈에서 특정 블록을 클릭하여 상세 화면에 진입한다
- **Then** 블록 목표 제목, 세부 실행 과제 목록, 각 셀의 완료 여부가 표시된다

### TC-05-2. 실행 과제 완료 토글
- **Given** 블록 상세 화면에서 미완료 상태의 실행 과제가 표시된 상태
- **When** 해당 실행 과제를 클릭한다
- **Then** 완료 상태로 변경되고 시각적 표시(체크 등)가 변경된다
- **When** 동일 과제를 다시 클릭한다
- **Then** 미완료 상태로 돌아간다

### TC-05-3. 뒤로 가기 → 홈 복귀
- **Given** 블록 상세 화면이 표시된 상태
- **When** 뒤로 가기(헤더 백 버튼)를 클릭한다
- **Then** 홈 화면으로 복귀한다

## TC-08. 설정 (Settings)

### TC-08-1. 설정 화면 기본 요소 확인
- **Given** 설정 화면으로 이동한다
- **Then** "앱 정보" 섹션에 버전(1.0.0), "데이터 관리" 섹션에 "데이터 초기화" 버튼이 표시된다

### TC-08-2. 데이터 초기화 - 취소
- **Given** 설정 화면에서 "데이터 초기화"를 클릭한다
- **When** 확인 다이얼로그에서 "취소"를 클릭한다
- **Then** 데이터가 초기화되지 않는다

### TC-08-3. 데이터 초기화 - 확인
- **Given** 설정 화면에서 "데이터 초기화"를 클릭한다
- **When** 확인 다이얼로그에서 "초기화"를 클릭한다
- **Then** 데이터가 초기 Mock 데이터로 복원된다
